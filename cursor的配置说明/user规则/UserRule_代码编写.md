# User Rule：代码编写与结构规范

## 规则内容

### 代码编写规范

```
代码编写规范：
- 使用详细的简体中文注释和文档字符串
- 函数和类必须有完整的类型提示
- 异常处理要完善，提供有用的错误信息
- 所有代码注释、文档、解释都使用简体中文
```

### 文件结构规范

```
文件结构规范：
- 遵循项目现有的目录结构
- 测试文件放在 "cursor_test" 文件夹下，文件名以 test_ 开头
- 代码相关文档放在 "cursor_docs" 文件夹下
```

### 代码生成规范

```
测试代码生成规则：
- 如果代码比较复杂，必须编写测试代码来验证主逻辑是否有问题
- 测试代码必须存放在 "cursor_test" 文件夹下
- 测试文件命名以 test_ 开头，如 test_*.py、test_*.class
- 测试应该覆盖核心业务逻辑和关键功能点

文档生成规则：
- 代码相关的文档需要自行判断是否要生成
- 如果代码逻辑复杂、涉及重要功能或需要说明使用方式，应该生成相应文档
- 文档文件存放在 "cursor_docs" 文件夹下
- 文档可以包括：API 文档、使用说明、架构设计说明等
```

## 使用说明

此规则定义了代码编写、文件结构和代码生成的所有规范。

**适用场景**：
- 编写新代码时
- 重构代码时
- 生成代码时
- 组织项目结构时

**配合使用**：
- 配合 `UserCommands_代码编写.md` 中的命令使用
- 规则定义规范，命令提供模板
