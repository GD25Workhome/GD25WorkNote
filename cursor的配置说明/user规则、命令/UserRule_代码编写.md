# User Rule：代码编写与结构规范

## 规则内容

### 代码编写规范

```
代码编写规范：
- 使用详细的简体中文注释和文档字符串
- 函数和类必须有完整的类型提示
- 异常处理要完善，提供有用的错误信息
- 所有代码注释、文档、解释都使用简体中文
```

### 文件结构规范

```
文件结构规范：
- 遵循项目现有的目录结构
- 测试文件放在项目根目录，文件名以 test_ 开头
```

### 代码生成规范

```
默认规则（不主动生成）：
- 除非明确要求，否则不要主动生成测试代码
- 不要创建 test_*.py、test_*.class 文件，除非用户明确要求
- 不要生成单元测试、集成测试等相关代码
- 除非明确要求，否则不要主动生成说明类的文档
- 不要创建新的 README.md、文档文件，除非用户明确要求
- 不要生成代码注释文档、API 文档等，除非用户明确要求

例外情况（应该生成）：
- 如果用户明确说"生成测试"、"创建文档"、"写文档"等，则应该生成
- 如果用户要求"完善文档"、"更新 README"等，则应该更新
```

## 使用说明

此规则定义了代码编写、文件结构和代码生成的所有规范。

**适用场景**：
- 编写新代码时
- 重构代码时
- 生成代码时
- 组织项目结构时

**配合使用**：
- 配合 `UserCommands_代码编写.md` 中的命令使用
- 规则定义规范，命令提供模板
